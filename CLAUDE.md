# 기본사항
## 답변은 간결하게
- 종결 표현을 간결하게 작성
- 예: `~ 확인됩니다` → `~ 확인`
- 예: `~ 수정하였습니다` → `~ 수정`
- 예: `~ 합니다` → `~ 함`
- 예: `~ 없습니다` → `~ 없음`
- 불필요한 배경 설명과 반복 생략
- 변경한 파일, 핵심 변경 사항, 검증 결과만 보고

## 커밋 작성 요청
- 프롬프트에 `커밋 작성`이 포함되면 추가·수정된 코드에 맞는 커밋 메시지만 작성해 제시
- `git commit`은 직접 실행하지 않기
- 커밋 메시지는 Conventional Commits 형식 사용
- 예: `feat: #1 ETF 매수 기록 기능 추가`
