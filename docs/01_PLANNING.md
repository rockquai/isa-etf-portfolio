## 2. 🧭 앱 지향점 (이상형)

### 4가지 핵심 가치

| 가치 | 의미 | 앱 내 구현 위치 |
|---|---|---|
| 🏆 마일스톤 | 3/5/10/20년 배당 목표 달성률 시각화 | GoalGauge + DividendTimeline |
| 💪 동기부여 | 나만의 다짐 문구 + 줍줍 타이밍 알림 | MyGoalBanner + Push 알림 |
| 🛒 소비 방지 | "장바구니 비우고 1주 사자" 브랜드 보이스 | 상황별 UX 카피 |
| 📅 습관 형성 | 매일 아침 루틴 (뉴스→AI→매수 기록) | 아침 루틴 통합 대시보드 |

### Before / After 사용자 변화

```
Before (앱 사용 전)
├── 네이버/다음 쇼핑 장바구니에 충동 소비
├── 재테크는 해야 하는데 뭘 해야 할지 모름
├── 적금은 귀찮고, 주식은 무섭고
└── 미래 계획 없이 오늘만 삶

After (앱 사용 후)
├── 장바구니 대신 ETF 1주 담는 습관
├── 매일 아침 경제 뉴스 + AI 브리핑으로 시장 감각
├── 3년 후 월 배당금이 눈에 보여서 꾸준히 지속
└── 불필요한 소비 줄고 배당 파이프라인 구축 중
```

## 6. 📊 배당 예측 대시보드 기획

### 핵심 UX 원리

```
기존 증권 앱  →  "수익률 +3.2%", "평가손익 -12,000원"
               오늘의 감정 자극 → 매도 충동

이 앱         →  "3년 후 월 8만원, 10년 후 월 23만원"
               미래 목표 자극 → 매수 습관 형성
```

### 타임라인 구성

| 기간 | 의미 |
|---|---|
| 3년 후 | 가까운 목표 — "3년은 할 수 있을 것 같다" 심리적 접근성 |
| 5년 후 | 중간 체크포인트 |
| 10년 후 | 장기 목표 확인 |
| 20년 후 | 은퇴/파이어족 시나리오 |

## 7. ✏️ 나만의 투자 다짐 배너 기획

### 리텐션 원리

```
앱이 정해준 카피      →  3일 후 식상 → 앱 삭제
내가 직접 쓴 목표    →  절대 식상해지지 않음 → 매일 접속
```

### 사용자 입력 예시

```
"다음/네이버 쇼핑 하지말고 1주 ETF 사고 배당 받아서 BTS 콘서트 가자!! 정신차려!"
"10년후 배당 받고 BTS 콘서트 가자!"
"30년 후 월 100만원 배당으로 제주도 한달살기"
```

### 줍줍 알림과 연동

```
📲 푸시 알림 예시

"BTS 콘서트 가려면 정신차려! 💜"    ← 내가 쓴 다짐 문구
TIGER가 평단가보다 낮아요. 지금이 줍줍 타이밍! 🛒→📈
```

### 입력 제약

- 최대 100자
- 줄바꿈 허용 (multi-line)
- 이모지 허용
- Supabase `user_settings.goal_message` 컬럼에 저장

## 11. 🔍 시니어 FE 검토 결과 — 기획 이슈

> **검토 일자:** 2025-06  
> **검토 관점:** 서버리스 배포 기준 포트폴리오 프로젝트 완성도

### P1 — 구현 전 확정 필요

#### 1. 배당 계산 가정값 사용자 노출 필요

현재 기획에서 "연 5% 배당 성장률"과 "매일 1주 매수 가정"이 고정값으로 숨어 있음.

```
문제
├── 20년 시뮬레이션에서 매 거래일(250일) × 20년 = 5,000주 추가 가정
├── 실제 사용자 매수 패턴과 크게 다를 수 있음
└── 면책 문구만으로는 "예측값이 너무 다르다"는 불만 방지 어려움

개선 방향
├── DividendTimeline 카드 하단에 가정값 요약 노출
│   예) "연 배당 성장률 5% · 매일 1주 추가 매수 가정"
├── 설정 화면에서 성장률 슬라이더 제공 (3% / 5% / 7% 선택)
└── "현재 보유 수량만으로 계산" vs "매일 1주 추가 가정" 토글 분리
```

#### 2. Free 티어 제약 범위 모호

```
현재 기획
├── ETF 종목 최대 3개
├── AI 브리핑 월 5회
└── 거래 내역 제한 없음 ← 미정의

문제
├── 종목이 3개인데 거래 내역은 무제한이면 TransactionHistory 비용 발생
└── Free 사용자가 종목 3개에 수천 건 거래 내역을 쌓는 케이스 대응 없음

확정 필요 항목
├── 거래 내역: 종목당 최근 N건 표시 (예: Free 50건, Pro 무제한)
└── 포트폴리오 페이지 진입 시 Free 배너 노출 여부
```

#### 3. 온보딩 플로우 미정의 (빈 상태)

```
현재 UX 플로우 (Section 5)는 ETF를 이미 보유한 사용자 기준

정의 필요한 빈 상태 시나리오
├── 신규 가입 → ETF 0종목 → 대시보드 진입 시 화면 구성
│   - DividendTimeline: 0원 표시? 빈 상태 카드? 숨김?
│   - GoalGauge: 0% 상태에서 목표 금액 입력 UI 필요
│   - AIBriefing: 보유 종목 없을 때 브리핑 내용?
│   - BuyRecordButton: 종목 없을 때 비활성화 or 포트폴리오 페이지 유도
└── 권장 UX: 온보딩 카드로 "첫 ETF 종목 추가하기 →" CTA 노출
```

#### 4. 목표 금액 입력 누락

```
GoalGauge가 "월 N만원 목표까지 M만원 남았어요"를 보여주려면
사용자가 목표 금액을 입력해야 함

현재 user_settings 테이블에 goal_message(문구)는 있으나
goal_amount(목표 배당액) 컬럼이 없음

→ Section 7 또는 설정 화면에 목표 금액 입력 UX 추가 필요
→ BE 스키마에 goal_monthly_amount INTEGER DEFAULT 500000 추가
```
