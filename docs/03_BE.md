## 3. 🗄️ Supabase 스키마

### ⚠️ P0 — 타임존 통일 (선행 필수)

**문제:** Vercel 서버리스는 UTC로 동작. 한국 아침 6~9시 = UTC 전날 21~24시  
→ 매수 기록이 **전날 날짜**로 저장 (현재 코드에서 `new Date().toISOString().slice(0, 10)` 사용)  
→ 루틴 완료 판정(날짜 기준)이 모두 1일씩 밀림

**해결:**
1. `lib/getKstDate.ts` 신규 작성:
```typescript
export function getKstDate(): string {
  return new Date(new Date().getTime() + 9 * 60 * 60 * 1000)
    .toISOString()
    .slice(0, 10)
}
```

2. `app/actions/transaction.ts` 수정:
```typescript
date: getKstDate(),  // UTC → KST
```

3. Supabase 스키마 `transactions.date` 기본값 수정:
```sql
ALTER TABLE transactions
  ALTER COLUMN date SET DEFAULT (NOW() AT TIME ZONE 'Asia/Seoul')::date;
```

### RLS (Row Level Security) 설정

```sql
-- 모든 테이블에 RLS 활성화
ALTER TABLE etf_holdings    ENABLE ROW LEVEL SECURITY;
ALTER TABLE transactions     ENABLE ROW LEVEL SECURITY;
ALTER TABLE user_settings    ENABLE ROW LEVEL SECURITY;
ALTER TABLE routine_stickers ENABLE ROW LEVEL SECURITY;
ALTER TABLE sticker_board_rewards ENABLE ROW LEVEL SECURITY;
ALTER TABLE term_cards       ENABLE ROW LEVEL SECURITY;

-- 본인 데이터만 접근 가능 정책
CREATE POLICY "users can manage own etf_holdings"
  ON etf_holdings FOR ALL
  USING (auth.uid() = user_id);

CREATE POLICY "users can manage own transactions"
  ON transactions FOR ALL
  USING (auth.uid() = user_id);

CREATE POLICY "users can manage own settings"
  ON user_settings FOR ALL
  USING (auth.uid() = user_id);

CREATE POLICY "users can read own routine_stickers"
  ON routine_stickers FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "users can insert own routine_stickers for today"
  ON routine_stickers FOR INSERT
  WITH CHECK (
    auth.uid() = user_id
    AND sticker_date = (NOW() AT TIME ZONE 'Asia/Seoul')::date
  );

CREATE POLICY "users can manage own sticker_board_rewards"
  ON sticker_board_rewards FOR ALL
  USING (auth.uid() = user_id);

CREATE POLICY "anyone can read term_cards"
  ON term_cards FOR SELECT
  USING (true);
```

```typescript
    .from('etf_holdings')
    .select('*')
    .eq('user_id', userId)
```

## 7. 🤖 LLM 파이프라인

### 단일 호출 구조 (Anthropic Claude Haiku 4.5)

```
generateETFBriefing(newsItems, holdings)
  입력: 뉴스 제목 배열 + 보유 ETF 데이터 (~500 토큰)
  모델: claude-haiku-4-5 (@anthropic-ai/sdk 공식 SDK)
  출력: ETF 맞춤형 브리핑 (~600 토큰)
  
  성공: { result: briefing, tier: 'ai' }
  실패 (API 오류): { result: MOCK_BRIEFING, tier: 'fallback' }

비용: ~$0.0035/회 (입력 $1 + 출력 $5 per 1M 토큰)
```

**설계 변경 (이전 → 현재):**
- ~~2단계 체이닝 (gpt-4o-mini → Claude Sonnet)~~ → **단일 호출 (Haiku 4.5)**
- ~~tier별 Mock/실제 분기~~ → **전원 실제 호출 + 월 5회 카운트 제한**
- ~~OpenAI/Anthropic 듀얼 API~~ → **Anthropic SDK만**

```typescript
// lib/llm-chain.ts
import Anthropic from '@anthropic-ai/sdk'

const client = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY,
})

// Mock 브리핑 — API 오류 시 폴백
export const MOCK_BRIEFING = `오늘 미국 고용지표가 예상보다 둔화되며 금리 인하 기대감이 높아졌습니다.
보유하신 TIGER 미국배당다우존스는 현재 평단가 대비 낮은 수준에 있어 분할매수를 고려해볼 만한 시점입니다.
장기 배당 파이프라인 관점에서 오늘 1주 적립은 좋은 선택일 수 있습니다.
※ 본 내용은 정보 제공용이며 투자 판단은 본인 책임입니다.`

// 메인 함수: 전원 실제 호출 (API 키 필수)
export async function generateETFBriefing(
  newsItems: Array<{ title: string }>,
  holdings: Array<{ ticker: string; avgPrice: number; currentPrice: number }>
): Promise<{ result: string; tier: 'ai' | 'fallback' }> {
  try {
    const newsText = newsItems.map(n => n.title).join('\n') || '오늘은 특별히 눈에 띄는 시장 뉴스가 없습니다.'
    const holdingContext = holdings
      .map(h => `${h.ticker}: 평단가 ${h.avgPrice.toLocaleString()}원 / 현재가 ${h.currentPrice.toLocaleString()}원`)
      .join('\n')

    const response = await client.messages.create({
      model: 'claude-haiku-4-5',
      max_tokens: 600,
      messages: [
        {
          role: 'user',
          content: `당신은 ETF 장기 투자 도우미입니다.
아래 정보를 바탕으로 오늘 아침 투자 브리핑을 작성해주세요.

[보유 ETF 현황]
${holdingContext || '(현재 보유 종목 없음)'}

[오늘의 경제 뉴스]
${newsText}

작성 규칙:
- 3~5문장으로 간결하게
- 주린이도 이해할 수 있는 쉬운 언어
- 반드시 마지막에 면책 문구 포함:
  "※ 본 내용은 정보 제공용이며 투자 판단은 본인 책임입니다."`,
        },
      ],
    })

    const briefing = response.content[0].type === 'text' ? response.content[0].text : MOCK_BRIEFING
    return { result: briefing, tier: 'ai' }
  } catch (error) {
    console.error('LLM chain failed, using mock fallback:', error)
    return { result: MOCK_BRIEFING, tier: 'fallback' }
  }
}
```

```typescript
// app/api/ai-briefing/route.ts
import { NextRequest, NextResponse } from 'next/server'
import { generateETFBriefing } from '@/lib/llm-chain'
import { supabase } from '@/lib/supabase'

const FREE_MONTHLY_LIMIT = 5  // 전체 사용자 월 5회 제한

export async function POST(req: NextRequest) {
  const { newsItems, holdings, userId } = await req.json()

  // 1. 호출 횟수 확인 (전원 동일 로직)
  const { data: settings } = await supabase
    .from('user_settings')
    .select('user_tier, ai_call_count, ai_call_reset_at')
    .eq('user_id', userId)
    .single()

  const userTier = settings?.user_tier ?? 'free'
  const callCount = settings?.ai_call_count ?? 0

  // 2. 월 초기화 여부 판정 (이번 달 1일 이후인지 확인)
  const resetDate = settings?.ai_call_reset_at ? new Date(settings.ai_call_reset_at) : null
  const currentMonthStart = new Date(new Date().getFullYear(), new Date().getMonth(), 1)
  const isNewMonth = !resetDate || resetDate < currentMonthStart

  let currentCount = callCount
  if (isNewMonth) {
    // ⚠️ 알려진 버그: 월 초기화 후 DB에 oldCount+1로 덮어써짐
    //    로컬 변수 currentCount가 갱신되지 않아 리셋이 무효화됨
    //    예: callCount=5 → 리셋(0) → currentCount는 여전히 5 → upsert → 6
    //    근본 원인: read-then-upsert 비원자적 처리 + 동시요청 레이스 컨디션
    //    개선: increment_ai_call() Postgres 함수 사용 또는 row locking
    await supabase.from('user_settings')
      .update({ ai_call_count: 0, ai_call_reset_at: new Date().toISOString() })
      .eq('user_id', userId)
    // currentCount = 0  ← 버그 회피: 로컬 변수를 0으로 명시적 설정
  }

  // 3. 월 한도 초과 시 차단 (tier 구분 없이 전원 동일)
  if (currentCount >= FREE_MONTHLY_LIMIT) {
    return NextResponse.json(
      {
        error: 'FREE_LIMIT_EXCEEDED',
        message: `이번 달 무료 제공 횟수(${FREE_MONTHLY_LIMIT}회)를 모두 사용했어요. 다음 달 1일에 자동으로 초기화됩니다.`,
        remaining: 0,
      },
      { status: 429 }
    )
  }

  // 4. AI 브리핑 생성 (전원 실제 호출, API 키 필수)
  const result = await generateETFBriefing(newsItems, holdings)

  // 5. 호출 횟수 증가
  await supabase.from('user_settings')
    .update({ ai_call_count: currentCount + 1 })
    .eq('user_id', userId)

  return NextResponse.json(result)
}
```

```typescript
export async function GET(req: NextRequest) {
```

#### 2. AI 호출 횟수 Race Condition

현재 카운트 로직이 `SELECT → 비교 → UPDATE` 3단계로 분리되어 동시 요청 시 제한 우회 가능.

```typescript
// ❌ 현재 — 비원자적 처리
const callCount = settings?.ai_call_count ?? 0
if (callCount >= FREE_MONTHLY_LIMIT) { /* 차단 */ }
await supabase.from('user_settings').update({ ai_call_count: callCount + 1 })
// → 동시 요청 2개가 동시에 callCount=4를 읽으면 둘 다 통과 후 5, 5로 업데이트

// ✅ 수정 — Supabase DB Function으로 원자적 처리
-- Supabase SQL Editor에서 실행
CREATE OR REPLACE FUNCTION increment_ai_call(p_user_id UUID, p_limit INT)
RETURNS TABLE(allowed BOOLEAN, new_count INT) AS $$
DECLARE
  v_count INT;
  v_tier TEXT;
  v_reset_at TIMESTAMPTZ;
BEGIN
  SELECT ai_call_count, user_tier, ai_call_reset_at
  INTO v_count, v_tier, v_reset_at
  FROM user_settings WHERE user_id = p_user_id FOR UPDATE;  -- 행 잠금

  -- 월 초기화
  IF v_reset_at < date_trunc('month', NOW()) THEN
    v_count := 0;
    UPDATE user_settings
    SET ai_call_count = 0, ai_call_reset_at = NOW()
    WHERE user_id = p_user_id;
  END IF;

  IF v_tier = 'pro' OR v_count < p_limit THEN
    UPDATE user_settings
    SET ai_call_count = v_count + 1
    WHERE user_id = p_user_id;
    RETURN QUERY SELECT TRUE, v_count + 1;
  ELSE
    RETURN QUERY SELECT FALSE, v_count;
  END IF;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

```typescript
// Route Handler에서 RPC 호출로 대체
```

#### 3. avg_price 자동 재계산 누락

`addTransaction`으로 매수를 기록해도 `etf_holdings.avg_price`가 갱신되지 않아 배당 계산이 부정확해짐.

```typescript
// ✅ 수정 — addTransaction 시 평단가 재계산 포함
export async function addTransaction(
  userId: string,
  transaction: { etf_id: string; price: number; quantity: number; date: string }
) {
  // 1. 현재 보유 수량과 평단가 조회
  const { data: holding } = await supabase
    .from('etf_holdings')
    .select('avg_price, quantity')
    .eq('id', transaction.etf_id)
    .eq('user_id', userId)
    .single()

  if (!holding) throw new Error('ETF holding not found')

  // 2. 가중 평균 단가 재계산
  const totalCost = holding.avg_price * holding.quantity + transaction.price * transaction.quantity
  const totalQuantity = holding.quantity + transaction.quantity
  const newAvgPrice = totalCost / totalQuantity

  // 3. 트랜잭션 INSERT + 평단가 UPDATE (동시)
  const [txResult] = await Promise.all([
    supabase.from('transactions').insert({ ...transaction, user_id: userId }).select().single(),
    supabase.from('etf_holdings')
      .update({
        avg_price: Math.round(newAvgPrice * 100) / 100,
        quantity: totalQuantity,
        current_price: transaction.price,  // 최신 매수가를 현재가로 업데이트
      })
      .eq('id', transaction.etf_id)
      .eq('user_id', userId),
  ])

  if (txResult.error) throw txResult.error
  return txResult.data
}
```

### ⚠️ 대응 1 — 비활성 일시정지 방지

```yaml
jobs:
  ping:
    runs-on: ubuntu-latest
    steps:
      - name: Ping Supabase
        run: |
          curl -s \
            "${{ secrets.NEXT_PUBLIC_SUPABASE_URL }}/rest/v1/user_settings?select=id&limit=1" \
            -H "apikey: ${{ secrets.NEXT_PUBLIC_SUPABASE_ANON_KEY }}" \
            -H "Authorization: Bearer ${{ secrets.NEXT_PUBLIC_SUPABASE_ANON_KEY }}"
```

### ⚠️ 대응 2 — DB 백업 (스키마 + 데이터)

**문제:** Free 플랜은 자동 DB 백업이 없음.
장기간 방치 후 프로젝트가 삭제되거나 실수로 테이블을 날리면 복구 불가.

#### 2-1. 스키마 백업 — `docs/schema.sql` 커밋

테이블 생성 SQL 전체를 `docs/schema.sql`로 Git에 보관.
프로젝트가 삭제되어도 5분 안에 재구성 가능.

```
docs/
├── schema.sql          # ← CREATE TABLE + RLS + DB Function 전체
├── 01_PLANNING.md
├── 02_FE.md
├── 03_BE.md
├── 04_FE_TASKS.md
└── 05_BE_TASKS.md
```

```sql
-- docs/schema.sql 구성 예시

-- 1. 테이블 생성
CREATE TABLE etf_holdings ( ... );
CREATE TABLE transactions ( ... );
CREATE TABLE user_settings ( ... );
CREATE TABLE ai_usage_logs ( ... );

-- 2. 컬럼 추가 (검토 반영)
ALTER TABLE etf_holdings ADD COLUMN is_profit BOOLEAN ...;
ALTER TABLE user_settings ADD COLUMN goal_monthly_amount INTEGER ...;
ALTER TABLE user_settings ADD COLUMN daily_purchase_enabled BOOLEAN ...;

-- 3. RLS 활성화
ALTER TABLE etf_holdings ENABLE ROW LEVEL SECURITY;
...

-- 4. RLS 정책
CREATE POLICY "users can manage own etf_holdings" ON etf_holdings ...;
...

-- 5. DB Function
CREATE OR REPLACE FUNCTION increment_ai_call(...) ...;
```

#### 2-2. Mock 데이터로 즉시 시연 가능한 구조 유지

DB가 날아가더라도 Mock 데이터 폴백으로 시연이 중단되지 않도록,
`lib/mock/` 데이터는 항상 실제 사용 가능한 상태로 유지할 것.

```
lib/mock/
├── etf.ts          # 5개 종목 Mock 데이터 — 항상 최신 상태 유지
├── transactions.ts  # 50개 거래 내역 Mock
├── news.ts          # [Mock] 접두사 뉴스 5개
└── briefing.ts      # Mock AI 브리핑 (면책 문구 포함)
```
