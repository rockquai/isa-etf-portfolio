## 3. 🗄️ Supabase 스키마

### ⚠️ P0 — 타임존 통일 (선행 필수)

**문제:** Vercel 서버리스는 UTC로 동작. 한국 아침 6~9시 = UTC 전날 21~24시  
→ 매수 기록이 **전날 날짜**로 저장 (현재 코드에서 `new Date().toISOString().slice(0, 10)` 사용)  
→ 루틴 완료 판정(날짜 기준)이 모두 1일씩 밀림

**해결:**
1. `lib/getKstDate.ts` 신규 작성:
```typescript
export function getKstDate(): string {
  return new Date(new Date().getTime() + 9 * 60 * 60 * 1000)
    .toISOString()
    .slice(0, 10)
}
```

2. `app/actions/transaction.ts` 수정:
```typescript
date: getKstDate(),  // UTC → KST
```

3. Supabase 스키마 `transactions.date` 기본값 수정:
```sql
ALTER TABLE transactions
  ALTER COLUMN date SET DEFAULT (NOW() AT TIME ZONE 'Asia/Seoul')::date;
```

```sql
  dividend_growth_rate NUMERIC(4,3) DEFAULT 0.050,  -- 5%

  created_at  TIMESTAMPTZ DEFAULT NOW()

-- 거래 내역
CREATE TABLE transactions (
  id        UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id   UUID REFERENCES auth.users(id) ON DELETE CASCADE,

  price     NUMERIC(10,2) NOT NULL,     -- 매수가
  quantity  INTEGER NOT NULL DEFAULT 1,

  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 루틴 완료 스티커 기록 (포도알)
CREATE TABLE routine_stickers (

  source        TEXT NOT NULL CHECK (source IN ('news_read','briefing_view','buy_record','watch_day')),
  created_at    TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE (user_id, sticker_date, source)   -- 멱등성: 중복 클릭 차단

-- 사용자 설정
CREATE TABLE user_settings (
  id                   UUID DEFAULT gen_random_uuid() PRIMARY KEY,

  created_at           TIMESTAMPTZ DEFAULT NOW()

-- AI 호출 로그 (비용 추적용)
CREATE TABLE ai_usage_logs (
  id            UUID DEFAULT gen_random_uuid() PRIMARY KEY,
```

## 7. 🤖 LLM 체이닝 파이프라인

### 2단계 체이닝 구조

```
[1단계] gpt-4o-mini
  입력: 뉴스 원문 전체 (~20,000 토큰)
  출력: 핵심 경제 키워드 3줄 요약 (~1,000 토큰)
  비용: ~$0.0036/회

    ↓ 압축된 요약문 전달

[2단계] Claude Sonnet
  입력: 뉴스 요약 + 유저 ETF 평단가 데이터 (~1,000 토큰)
  출력: 맞춤형 ETF 매수/매도 브리핑 (~500 토큰)
  비용: ~$0.0105/회

총 비용: ~$0.0141/회 (단일 Sonnet 대비 약 80% 절감)
```

```typescript
// lib/llm-chain.ts

// 1단계: 뉴스 요약 (저비용 모델)
async function summarizeNews(newsText: string): Promise<string> {
  const response = await fetch('https://api.openai.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${process.env.OPENAI_API_KEY}`,
    },
    body: JSON.stringify({
      model: 'gpt-4o-mini',
      messages: [
        {
          role: 'system',
          content: `당신은 경제 뉴스 요약 전문가입니다.
주어진 뉴스를 ETF 투자자에게 관련된 핵심 내용만
3줄 이내로 요약하세요. 반드시 한국어로 답변하세요.`,
        },
        { role: 'user', content: newsText },
      ],
      max_tokens: 300,
    }),
  })
  const data = await response.json()
  return data.choices[0].message.content
}

// 2단계: ETF 브리핑 (고성능 모델)
async function generateBriefing(
  newsSummary: string,
  holdings: Array<{ ticker: string; avgPrice: number; currentPrice: number }>
): Promise<string> {
  const holdingContext = holdings
    .map(h => `${h.ticker}: 평단가 ${h.avgPrice.toLocaleString()}원 / 현재가 ${h.currentPrice.toLocaleString()}원`)
    .join('\n')

  const response = await fetch('https://api.anthropic.com/v1/messages', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': process.env.ANTHROPIC_API_KEY!,
      'anthropic-version': '2023-06-01',
    },
    body: JSON.stringify({
      model: 'claude-sonnet-4-6',
      max_tokens: 500,
      messages: [
        {
          role: 'user',
          content: `당신은 ETF 장기 투자 도우미입니다.
아래 정보를 바탕으로 오늘 아침 투자 브리핑을 작성해주세요.

[보유 ETF 현황]
${holdingContext}

[오늘의 경제 뉴스 요약]
${newsSummary}

작성 규칙:
- 3~5문장으로 간결하게
- 주린이도 이해할 수 있는 쉬운 언어
- 반드시 마지막에 면책 문구 포함:
  "※ 본 내용은 정보 제공용이며 투자 판단은 본인 책임입니다."`,
        },
      ],
    }),
  })
  const data = await response.json()
  return data.content[0].text
}

// Mock 브리핑 — Free 티어 또는 API 오류 시 사용
export const MOCK_BRIEFING = `오늘 미국 고용지표가 예상보다 둔화되며 금리 인하 기대감이 높아졌습니다.
보유하신 TIGER 미국배당다우존스는 현재 평단가 대비 낮은 수준에 있어 분할매수를 고려해볼 만한 시점입니다.
장기 배당 파이프라인 관점에서 오늘 1주 적립은 좋은 선택일 수 있습니다.
※ 본 내용은 정보 제공용이며 투자 판단은 본인 책임입니다.`

// 메인 체이닝 함수
export async function generateETFBriefing(
  newsItems: Array<{ title: string }>,
  holdings: Array<{ ticker: string; avgPrice: number; currentPrice: number }>,
  userTier: 'free' | 'pro'
): Promise<{ result: string; tier: string }> {

  // Free 티어: Mock 브리핑 반환 (비용 $0)
  if (userTier === 'free') {
    return { result: MOCK_BRIEFING, tier: 'free' }
  }

  // Pro 티어: 실제 LLM 체이닝
  const newsText = newsItems.map(n => n.title).join('\n')
  const summary  = await summarizeNews(newsText)
  const briefing = await generateBriefing(summary, holdings)
  return { result: briefing, tier: 'pro' }
}
```

```typescript
// app/api/ai-briefing/route.ts
import { NextRequest, NextResponse } from 'next/server'
import { generateETFBriefing } from '@/lib/llm-chain'
import { supabase } from '@/lib/supabase'

const FREE_MONTHLY_LIMIT = 5  // 무료 사용자 월 5회 제한

export async function POST(req: NextRequest) {
  const { newsItems, holdings, userId } = await req.json()

  // 1. 유저 티어 및 호출 횟수 확인
  const { data: settings } = await supabase
    .from('user_settings')
    .select('user_tier, ai_call_count, ai_call_reset_at')
    .eq('user_id', userId)
    .single()

  const userTier = settings?.user_tier ?? 'free'
  const callCount = settings?.ai_call_count ?? 0

  // 2. Free 티어 월 한도 초과 시 차단
  const isNewMonth = new Date(settings?.ai_call_reset_at) < new Date(
    new Date().getFullYear(), new Date().getMonth(), 1
  )

  if (isNewMonth) {
    // 월 초기화
    await supabase.from('user_settings')
      .update({ ai_call_count: 0, ai_call_reset_at: new Date().toISOString() })
      .eq('user_id', userId)
  } else if (userTier === 'free' && callCount >= FREE_MONTHLY_LIMIT) {
    return NextResponse.json(
      {
        error: 'FREE_LIMIT_EXCEEDED',
        message: `무료 플랜은 월 ${FREE_MONTHLY_LIMIT}회까지 이용 가능합니다.`,
        remaining: 0,
      },
      { status: 429 }
    )
  }

  // 3. LLM 체이닝 실행
  const result = await generateETFBriefing(newsItems, holdings, userTier)

  // 4. 호출 횟수 증가
  await supabase.from('user_settings')
    .update({ ai_call_count: callCount + 1 })
    .eq('user_id', userId)

  return NextResponse.json(result)
}
```

```typescript
// app/api/portfolio/route.ts
import { NextRequest, NextResponse } from 'next/server'
import { supabase } from '@/lib/supabase'

// ETF 목록 조회
export async function GET(req: NextRequest) {
  const userId = req.nextUrl.searchParams.get('userId')
  if (!userId) return NextResponse.json({ error: 'userId required' }, { status: 400 })

  const { data, error } = await supabase
    .from('etf_holdings')
    .select('*')
    .eq('user_id', userId)
    .order('created_at')

  if (error) return NextResponse.json({ error }, { status: 500 })
  return NextResponse.json({ data })
}

// ETF 종목 추가
export async function POST(req: NextRequest) {
  const { userId, ticker, avgPrice, quantity, annualDividendPerShare } = await req.json()

  const { data, error } = await supabase
    .from('etf_holdings')
    .insert({
      user_id: userId,
      ticker,
      avg_price: avgPrice,
      current_price: avgPrice,   // 초기값은 평단가와 동일
      quantity,
      annual_dividend_per_share: annualDividendPerShare,
    })
    .select()
    .single()

  if (error) return NextResponse.json({ error }, { status: 500 })
  return NextResponse.json({ data })
}

// ETF 종목 삭제
export async function DELETE(req: NextRequest) {
  const { etfId, userId } = await req.json()

  const { error } = await supabase
    .from('etf_holdings')
    .delete()
    .eq('id', etfId)
    .eq('user_id', userId)   // 본인 데이터만 삭제 가능

  if (error) return NextResponse.json({ error }, { status: 500 })
  return NextResponse.json({ success: true })
}
```

## 9. 💰 LLM 비용 계산 (README 기재용)

### 단일 Sonnet 구조 vs 체이닝 구조 비교

| 구조 | 모델 | Input 토큰 | Output 토큰 | 비용 |
|---|---|---|---|---|
| **단일 Sonnet** | Claude Sonnet | 20,000 ($0.003/1K) | 500 ($0.015/1K) | **$0.0675** |
| **체이닝 1단계** | gpt-4o-mini | 20,000 ($0.00015/1K) | 1,000 ($0.0006/1K) | $0.0036 |
| **체이닝 2단계** | Claude Sonnet | 1,000 ($0.003/1K) | 500 ($0.015/1K) | $0.0105 |
| **체이닝 합계** | | | | **$0.0141** |

**절감율: (1 - 0.0141 / 0.0675) × 100 ≈ 79.1% → 약 80% 절감**

#### 2. AI 호출 횟수 Race Condition

현재 카운트 로직이 `SELECT → 비교 → UPDATE` 3단계로 분리되어 동시 요청 시 제한 우회 가능.

```typescript
// ❌ 현재 — 비원자적 처리
const callCount = settings?.ai_call_count ?? 0
if (callCount >= FREE_MONTHLY_LIMIT) { /* 차단 */ }
await supabase.from('user_settings').update({ ai_call_count: callCount + 1 })
// → 동시 요청 2개가 동시에 callCount=4를 읽으면 둘 다 통과 후 5, 5로 업데이트

// ✅ 수정 — Supabase DB Function으로 원자적 처리
-- Supabase SQL Editor에서 실행
CREATE OR REPLACE FUNCTION increment_ai_call(p_user_id UUID, p_limit INT)
RETURNS TABLE(allowed BOOLEAN, new_count INT) AS $$
DECLARE
  v_count INT;
  v_tier TEXT;
  v_reset_at TIMESTAMPTZ;
BEGIN
  SELECT ai_call_count, user_tier, ai_call_reset_at
  INTO v_count, v_tier, v_reset_at
  FROM user_settings WHERE user_id = p_user_id FOR UPDATE;  -- 행 잠금

  -- 월 초기화
  IF v_reset_at < date_trunc('month', NOW()) THEN
    v_count := 0;
    UPDATE user_settings
    SET ai_call_count = 0, ai_call_reset_at = NOW()
    WHERE user_id = p_user_id;
  END IF;

  IF v_tier = 'pro' OR v_count < p_limit THEN
    UPDATE user_settings
    SET ai_call_count = v_count + 1
    WHERE user_id = p_user_id;
    RETURN QUERY SELECT TRUE, v_count + 1;
  ELSE
    RETURN QUERY SELECT FALSE, v_count;
  END IF;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```


### P1 — 구현 시 막히는 것들

#### 3. avg_price 자동 재계산 누락

`addTransaction`으로 매수를 기록해도 `etf_holdings.avg_price`가 갱신되지 않아 배당 계산이 부정확해짐.

```typescript
// ✅ 수정 — addTransaction 시 평단가 재계산 포함
export async function addTransaction(
  userId: string,
  transaction: { etf_id: string; price: number; quantity: number; date: string }
) {
  // 1. 현재 보유 수량과 평단가 조회
  const { data: holding } = await supabase
    .from('etf_holdings')
    .select('avg_price, quantity')
    .eq('id', transaction.etf_id)
    .eq('user_id', userId)
    .single()

  if (!holding) throw new Error('ETF holding not found')

  // 2. 가중 평균 단가 재계산
  const totalCost = holding.avg_price * holding.quantity + transaction.price * transaction.quantity
  const totalQuantity = holding.quantity + transaction.quantity
  const newAvgPrice = totalCost / totalQuantity

  // 3. 트랜잭션 INSERT + 평단가 UPDATE (동시)
  const [txResult] = await Promise.all([
    supabase.from('transactions').insert({ ...transaction, user_id: userId }).select().single(),
    supabase.from('etf_holdings')
      .update({
        avg_price: Math.round(newAvgPrice * 100) / 100,
        quantity: totalQuantity,
        current_price: transaction.price,  // 최신 매수가를 현재가로 업데이트
      })
      .eq('id', transaction.etf_id)
      .eq('user_id', userId),
  ])

  if (txResult.error) throw txResult.error
  return txResult.data
}
```

```yaml
            "${{ secrets.NEXT_PUBLIC_SUPABASE_URL }}/rest/v1/user_settings?select=id&limit=1" \
            -H "apikey: ${{ secrets.NEXT_PUBLIC_SUPABASE_ANON_KEY }}" \
```
